# Premium and Institutional Access Plan

Premium access should add workflow depth on top of the trusted public directory. It must not sell factual corrections, score improvement, or paid ranking placement.

## Public baseline

Keep these public:

- source-backed contractor profiles
- core public signals and caveats
- source links and freshness metadata
- correction request paths for source-backed facts
- clear separation between USAspending facts, curated overlays, and contractor-provided context

## Candidate paid layers

### Professional workflow tools

- saved searches and watchlists
- alerts for new awards, concentration changes, and freshness events
- deeper signal history and comparison views
- private notes and team workspaces

### Claimed profile management

- verified profile context management
- capabilities, certifications, contact routing, and public business-development context
- review workflow visibility for submitted updates

Corrections to source-backed public facts remain outside the paid value proposition.

### Teaming discovery tools

- advanced partner filters
- shortlist exports
- alerting for new potential partners
- declared capability matching after verification
- relationship and activity explanations based on public records

### Institutional products

- bulk exports
- recurring datasets
- custom reports
- portfolio and market monitoring
- analyst-ready methodology documentation

## Access-control principles

- Public pages remain indexable and useful.
- Paid access expands history, workflow, alerts, exports, and collaboration.
- Paid customers cannot buy better signal status or suppress source-backed facts.
- Contractor-declared content is labeled separately from public-data signals.

## Deferred implementation questions

- billing provider and entitlement storage
- team/workspace model
- export rate limits and licensing terms
- institutional report packaging
- admin approval workflows for claimed context
